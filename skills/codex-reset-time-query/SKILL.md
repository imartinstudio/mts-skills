# Codex 重置时间查询

用于帮助用户确认 Codex 相关额度、限流、任务冷却或订阅权益的重置/到期时间。不要凭记忆猜政策；时间必须来自当前可验证来源。

## 范围

适用：

- 用户问 Codex 何时重置、恢复、到期、刷新或解除限制。
- 用户提供了 Codex 错误消息、状态提示、截图、日志或账号页面信息。
- 需要把 UTC、相对时间或 epoch 时间换算成用户本地时区。

不适用：

- 查询 OpenAI API 账单、模型价格或普通 API rate limit，除非用户明确说它影响 Codex。
- 绕过限流、规避订阅限制、共享账号或提取 cookie/token。
- 没有任何可验证来源时给出确定时间。

## 输入

优先收集最小必要信息：

- Codex 表面：CLI、IDE 扩展、Codex 桌面应用、Web/Cloud、ChatGPT 中的 Codex。
- 原始提示：错误文本、状态栏、任务页面、用量页面或日志中的完整时间字段。
- 用户时区；若未知，使用当前会话时区并明确说明。
- 是否允许使用浏览器/连接器查看已登录的官方账号页面。

## 工作流

1. 先确认查询目标：是“账号/订阅额度重置”、单个任务冷却、rate limit reset，还是试用/权益到期。
2. 从用户提供的原文、当前 Codex UI、CLI 输出、官方 OpenAI/Codex 文档或已授权账号页面中读取时间。涉及最新政策时，只使用官方 OpenAI 来源或当前产品界面。
3. 如果时间是相对描述，例如 `in 2 hours`，用当前日期和时区换算成绝对时间；如果是 UTC/ISO/epoch，转换为用户时区并保留原始值。
4. 若来源只说明“每日/每周/周期性重置”而没有账号专属时间，说明这是政策级信息，不等于该用户账户的精确重置点。
5. 如果找不到可验证时间，给出已检查的来源，并说明下一步需要用户提供哪条提示、截图或授权页面。

## 输出

用简洁格式回复：

- 结论：具体重置/到期时间，包含日期、时间和时区。
- 来源：来自错误消息、UI 页面、CLI 输出、官方文档或用户提供材料。
- 可信度：账号专属时间、政策推断或无法确认。
- 注意：若有多个限制，分别列出，不要把任务冷却和账户额度混为一谈。

## 安全与隐私

- 不要求用户发送密码、cookie、session token、完整请求头或支付信息。
- 截图和日志中出现邮箱、账单、令牌或组织 ID 时，提醒用户可先打码。
- 不访问非官方站点来确认账号权益；需要外部信息时优先官方 OpenAI 域名。
- 不承诺提额、解封或规避等待时间，只帮助定位和解释重置时间。
