# X @php_martin Appeal

Use this skill only for `@php_martin`. Keep its facts, identity, and evidence separate from every other X account.

## Evidence Discipline

1. Read [references/evidence-profile.md](references/evidence-profile.md) before drafting or analyzing.
2. Label material as **verified from 163 mail**, **user-provided**, or **hypothesis**. Do not upgrade a user-provided record into independently verified mail.
3. Prefer current X notices in the `@163.com` mailbox. If the mailbox is not connected to a mail connector, use only an existing authenticated 163-mail browser session after the user asks to inspect it; never request, read, or store passwords, verification codes, or cookies.
4. Attribute creator identity, original educational content, manual posting, and the absence of automation only when the user confirms those facts in the current request or they are verified by supplied evidence.
5. Treat VPN, IP, device, and location signals as possible risk indicators, not proof of a cause.
6. Never include an email address, phone number, government ID, password, reset code, or other sensitive identifier in a reference file or draft unless the user explicitly requests it for that specific draft.

## Workflow

1. Confirm the target handle is `@php_martin`.
2. Refresh X notices in the 163 mailbox when access is available. Record the date, sender, status, and exact decision; exclude unrelated X notifications.
3. Classify the account state: initial suspension, appeal pending, appeal denied, or remediation available on the X login page.
4. Select the requested draft mode:
   - **Initial appeal:** concise facts, good-faith request, and a request for review.
   - **Follow-up:** reference a prior case without repeating unsupported claims; ask for a manual review.
   - **Post-denial appeal:** ask X to identify the relevant activity/date and any available remediation.
5. Generate a concise English draft, Chinese reference, and evidence summary. Keep placeholders for personal contact details.
6. Stop after providing text. Do not submit a form, send an email, create a new account, or alter X settings.

## Drafting Rules

- Request a manual human review; never state that one is guaranteed.
- Be specific about evidence but avoid arguments based only on speculation.
- When confirmed by the user, describe the account as a real user sharing original educational AI/technology content. Do not call it an “empty account” or say it has never posted unless that is true for this account.
- Do not reuse claims from `@iMartinstudio`, including its no-post/no-reply history or Gmail timeline.
- Do not submit repeated, identical appeals. Make a follow-up materially different by requesting the missing factual basis or remediation path.
- If the current account UI says the account is permanently read-only while X's notice only offers generic on-screen remediation, include that discrepancy in a later appeal. State only the observed facts: no actionable remediation was displayed and the account cannot post, repost, or like.

## 163-Mail Handling

For requests to read the 163 mailbox:

1. Search for X notices from `notify@x.com`, `verify@x.com`, and `info@x.com` when the available mail surface supports it.
2. Read the full body of the newest suspension or appeal-decision notice before drafting.
3. Do not reproduce one-time codes, account-recovery links, or sensitive headers in the response.
4. State the date coverage and whether the mailbox was actually refreshed.

## Output Checklist

- State current status first.
- Separate evidence from inference.
- Include an English draft and Chinese reference unless the user asks for one language.
- Preserve contact fields as placeholders by default.
- Confirm that nothing was submitted.
